## Старт сессии

Прочитай **`docs/HISTORY.md`** (последние ~200 строк при большом файле). Если файла или каталога **`docs/`** нет — создай **`docs/HISTORY.md`** с кратким заголовком и веди журнал дальше.После **каждой** завершённой задачи дописывай: что сделано, почему, какие файлы затронуты, решения и компромиссы. Записывай **до коммита**. При изменениях GRACE указывай пути **`grace/**/*.xml`** и при необходимости **`grace/README.md`**.

Перед существенными правками следуй **`grace-core.md`** и связанным skills. **Расположение XML и прочих артефактов GRACE** — по **фактической структуре репозитория** и по таблице «шаблон → целевой файл» в **`grace-init`**. 

- **Журнал:** после задачи дополни принятый в проекте файл истории до коммита (если журнал используется): что сделано, почему, файлы, компромиссы. Если правился GRACE — перечисли **реальные пути** всех затронутых артефактов и **`AGENTS.md`**, если менялась карта проекта.
- **Git и версия:** после задачи — **`git add`**, **`git commit`**, **`git push`** ; не откладывай коммит инструкцией пользователю. Номер версии — только из корневого **`VERSION`** (SemVer), остальные манифесты в том же коммите синхронизируй. 
- **Subject коммита:** `X.Y.Z: краткое описание` (номер **в начале** строки, как в **`VERSION`**).

- **Терминал и язык:** по умолчанию **PowerShell** или **`pwsh`**; не используй **`bash`**, **`sh`**, **`cmd`**, где есть эквивалент PowerShell, кроме явной просьбы пользователя или чтения чужих артефактов репозитория (например CI). Общение с пользователем — **по-русски**, если не попросили иначе.

## Уточнение к «Use tools for facts» (`grace-core.md`)  Запрет локального запуска приложения для проверки

**Не** поднимай и **не** прогоняй приложение (или его части) в терминале **ради проверки** в рантайме (Node/npm, PHP-серверы, Python dev-серверы, `docker compose up` для приложения и т.п.). Проверка: линтеры, статический анализ, чтение кода и контрактов. **Исключение:** пользователь **явно** просит конкретную команду запуска.
